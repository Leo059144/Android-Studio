Schéma de l'architecture applicative

L'application AppliMenu suit une architecture en couches : une couche présentation (les 4 activités Android), une couche accès aux données (BdAdapter + CreateBdEchantillon) et une couche données (base SQLite gsb.db).

Les flèches en tirets représentent la navigation par Intent. Les flèches pleines représentent les appels aux méthodes de la couche données.

Modèle de données

MCD — Modèle Conceptuel de Données

L'application gère une seule entité : ECHANTILLON. Il n'y a pas de relation entre entités car la base ne contient qu'une table.

MLD — Modèle Logique de Données

Passage du MCD au MLD (modèle relationnel) :

echantillons (_id, CODE, LIB, STOCK)
Attribut MCD Colonne SQL Type Contrainte Remarque
code (identifiant) _id + CODE INTEGER / TEXT PK AUTOINCREMENT / NOT NULL _id = clé technique SQLite ; CODE = identifiant métier
libelle LIB TEXT NOT NULL Nom du produit échantillon
quantiteStock STOCK TEXT NOT NULL Quantité en stock (valeur numérique stockée en TEXT)

Script SQL de création

CREATE TABLE echantillons (
    _id   INTEGER PRIMARY KEY AUTOINCREMENT,
    CODE  TEXT NOT NULL,
    LIB   TEXT NOT NULL,
    STOCK TEXT NOT NULL
);

1. Description de la base de données

Structure de la base de données SQLite

La base de données locale (fichier gsb.db) est stockée sur l'appareil Android. Elle est gérée par la classe CreateBdEchantillon qui hérite de SQLiteOpenHelper. Elle contient une seule table : echantillons.

Table : echantillons

Colonne Type SQL Contraintes Description Exemple
_id INTEGER PRIMARY KEY AUTOINCREMENT Identifiant unique auto-incrémenté 1, 2, 3…
CODE TEXT NOT NULL Code de l'échantillon code1
LIB TEXT NOT NULL Libellé / nom du produit lib1
STOCK TEXT NOT NULL Quantité en stock 3

Exécuté dans la méthode onCreate() de CreateBdEchantillon lors du premier lancement de l'application :

Documentation utilisateur

Ce guide décrit comment utiliser l'application AppliMenu pour gérer le stock d'échantillons de compléments alimentaires.

Lancement de l'application

Appuyez sur l'icône AppliMenu sur l'écran d'accueil de la tablette ou du téléphone Android. L'écran principal s'affiche avec trois boutons de navigation et un menu en haut à droite (⋮).

Écran 1 — Menu principal

Trois boutons permettent d'accéder aux fonctionnalités :

BoutonAction
Ajouter un nouvel échantillonOuvre le formulaire d'ajout (Écran 2)
Liste des échantillonsAffiche tous les échantillons en stock (Écran 3)
Maj d'un echantillonPermet de modifier le stock d'un produit (Écran 4)

Le menu ⋮ en haut à droite propose les mêmes actions ainsi que Quitter pour fermer l'application.

Écran 2 — Ajouter un échantillon

Pour ajouter un nouvel échantillon :

  • Saisissez le code du produit (ex : code1)
  • Saisissez le libellé (nom du produit)
  • Saisissez la quantité en stock (nombre entier)
  • Appuyez sur Ajouter

Un message de confirmation affiche le code, le libellé et la quantité si l'insertion est réussie. Si un champ est vide, le message « Veuillez remplir tous les champs. » s'affiche. Appuyez sur Quitter pour revenir au menu principal.

Écran 3 — Liste des échantillons

Cet écran affiche automatiquement la liste complète de tous les échantillons enregistrés en base de données. Chaque ligne affiche dans l'ordre : le code, le libellé et la quantité en stock.

Appuyez sur Quitter pour revenir au menu principal.

Écran 4 — Mise à jour du stock

Pour modifier le stock d'un échantillon existant :

  • Saisissez le code de l'échantillon à modifier
  • Saisissez la quantité à ajouter ou soustraire
  • Appuyez sur Ajouter pour augmenter le stock
  • Appuyez sur Supprimer pour diminuer le stock

Messages possibles :
— « Veuillez remplir tous les champs. » : un champ est vide
— « La quantité doit être un nombre. » : valeur non numérique saisie
— « Echantillon introuvable pour le code : xxx » : code absent de la base
— « quantité ajoutée » / « quantité supprimée » : mise à jour réussie

Fichiers Java (à quoi ils servent)

Fichier Java Hérite de Rôle
MainActivity.java AppCompatActivity Activité principale (point d'entrée de l'application). Crée le menu ActionBar depuis menu/main.xml, gère les clics des 3 boutons et des items du menu via des Intents explicites. Appelle jeuEssaiBd() au démarrage pour insérer des données de test.
AjoutEchantillon.java Activity Activité d'ajout d'un échantillon. Valide que les 3 champs (code, libellé, stock) ne sont pas vides. Crée un objet Echantillon et l'insère en base via BdAdapter.insererEchantillon(). Affiche un message de confirmation ou d'erreur.
ListeEchantillon.java Activity Activité de consultation. Récupère tous les enregistrements via BdAdapter.getData() (retourne un Cursor), puis utilise un SimpleCursorAdapter pour relier le Cursor au ListView. Chaque ligne utilise le template list_entree.xml.
MajEchantillon.java Activity Activité de mise à jour du stock. Recherche un échantillon par code (getEchantillonWithLib()), additionne ou soustrait la quantité saisie au stock actuel, puis met à jour l'enregistrement via BdAdapter.updateEchantillon().
Echantillon.java — Classe modèle (POJO). Représente un échantillon. Attributs : code, libelle, quantiteStock. Fournit deux constructeurs et des getters/setters.
CreateBdEchantillon.java SQLiteOpenHelper Gère la création et la migration de la base gsb.db. onCreate() exécute le script CREATE TABLE. onUpgrade() supprime et recrée la table.
BdAdapter.java — Couche d'accès aux données (DAO). Encapsule toutes les opérations CRUD sur la table echantillons : open(), close(), insererEchantillon(), getData(), getEchantillonWithLib(), updateEchantillon(), removeEchantillonWithCode().
